What problems it aims to solve?


Provides full DevTools for the (small) screen.


Debugging directly off the device/phone/tabled, see what happens on the device, no guessing, no alerts

Built-in mobile simulators



Browser built-in mobile simulators are good, but none replicates phone 1:1

Requirements



  • phone → Android or iOS
  • browser → Chrome, Firefox, Vivaldi, Safari
  • cable, maybe
  • weinre, maybe

Supported browsers and systems


Android

  • Chrome & Vivaldi
  • Firefox

iOS

  • Safari

Windows Phone

  • over the weinre

Chrome


All devices running Blink, no iOS

  • On the device: Android settings → Developer options → Enable USB debugging
  • On the browser: DevTools → Main menu → More tools → Remote devices
  • Not done yet: DevTools → Settings → Discover USB devices

Then, in Remote devices tab, open the page → Inspect, you're done!

Firefox


All devices running Gecko, no iOS

  • On the device: Android settings → Developer options → Enable USB debugging
  • On the device browser: Settings → Advanced → Remote debugging via USB/Wi-Fi
  • On the browser: Tools → Web Developer → WebIDE

For debugging via Wi-Fi, scan the QR code provided by WebIDE.

Vivaldi



Already covered, Vivaldi uses Blink rendering engine from Chromium project, everything is exactly the same as in Chromium/Chrome.

Safari


All iOS devices running iOS 6+ and Safari

  • On the device: Settings → Safari → Advanced → Web Inspector
  • On the browser: Preferences → Advanced → Show Develop menu in menu bar
  • Not there yet: Safari → Develop → Phone

Works with Xcode phone simulator as well, at least sometimes.
